# 🪐 Aetheria3D (VoxStory Engine)

**Aetheria3D** 是一套基于生成式 AI 与 Raylib 库的 3D 游戏场景自动化构建与开发系统。它旨在打破“连续语义”与“离散工程数据”的壁垒，实现从**自然语言文本**直接到**可游玩 3D 场景**的端到端生成。

## 🎯 项目目标与应用场景

传统 3D 游戏开发面临着资产创作成本极高、程序化生成（PCG）规则编写繁琐、以及关卡设计高度依赖人力的“不可能三角”。本项目通过整合大语言模型（LLM）、视觉生成模型（SDXL/3DGS）与轻量级桌面引擎，彻底重塑 3D 内容的生产流。

### 核心应用场景
1. **独立游戏敏捷开发**：为缺乏庞大美术团队的小型工作室提供“文本到可交互 3D 场景”的快速原型验证工具。
2. **沙盒与叙事游戏自动化量产**：实现基于玩家输入或小说文本，全自动生成带有独特地形地貌、碰撞物理和剧情节点的 3D 箱庭世界。
3. **AI 驱动的 UGC 平台底座**：作为未来玩家生成内容（User Generated Content）的底层技术支撑，让普通玩家通过自然语言即可构建可游玩的 3D 关卡。

## 📦 核心交付物与功能边界

### 1. 离线 AI 资产编译管线（The AI Compiler - Python）
支持接收自然语言文本，自动调度 SDXL+ControlNet 进行严格等距视角 2D 出图，并调用 DiffusionGS 极速生成 3DGS 点云文件（`.ply`）。包含一套将连续 3DGS 点云“强制体素化/网格化”的后处理脚本，以提取可用于物理碰撞的离散空间数据。

### 2. 轻量级 3D 桌面引擎（The Moonbit Client - Moonbit + Raylib）
提供一套用 Moonbit 语言编写、通过 FFI 深度绑定 Raylib 框架的桌面端游戏运行时。支持加载生成的 3D 资产、解析 LLM 输出的结构化 JSON（世界观、剧情节点），并提供基础的摄像机漫游、实体碰撞与 UI 交互。

> **⚠️ 功能边界限制**：本期交付物聚焦于“静态 3D 环境的生成与基础游玩逻辑”，暂不包含复杂的流体物理模拟、高级骨骼动画自动生成或大规模无缝开放世界动态加载。

## ⚙️ 预期交互流程

1. **创作阶段（离线）**：开发者在终端输入场景描述（如：“一个建立在雪山峡谷间的废弃矿镇，包含三个剧情探索点”）。
2. **编译阶段（自动化）**：AI 管线运行，LLM 输出包含坐标与剧情的 `world_data.json`；视觉管线输出对应的 3D 地形模型文件。
3. **游玩阶段（实时）**：启动 Moonbit 编译的桌面端 `.exe` 程序，引擎自动读取数据文件夹，渲染出带有物理碰撞的 3D 场景，玩家可通过键盘鼠标漫游，触碰剧情节点弹出交互 UI。

## 🏗️ 技术架构与选型

系统采用**“计算与渲染解耦”的 C/S 变体架构**，分为高算力的离线生产端和高能效的本地运行端：

- **大语言模型 (LLM)**：充当“世界架构师”，将模糊叙事转化为结构化的 JSON 拓扑图，确保生成的物理世界 100% 服务于剧情逻辑。
- **空间约束层 (SDXL + ControlNet)**：强制锁定等距视角（Isometric）或深度图，从源头消除透视畸变，提供完美的几何底座。
- **3D 生成层 (DiffusionGS)**：针对大范围场景切片重建速度极快，适合游戏箱庭地块的快速量产。
- **逻辑与渲染层 (Moonbit + Raylib)**：Moonbit 提供极速编译和内存安全的业务逻辑；Raylib 作为轻量级 C 图形库，作为 FFI 底层支撑，避免重型引擎（如 UE/Unity）带来的管线臃肿。
