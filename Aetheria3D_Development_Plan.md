# Aetheria3D (VoxStory Engine) 开发计划 (Updated)

## 概述
基于项目需求更新：
1. **优先级倒置**：优先完成 AI 无关的后半段任务（引擎核心、渲染、物理、UI），AI 管线任务延后并由开发者主导迭代。
2. **技术栈收敛**：尽量使用 Moonbit 替代 Python 完成数据处理与管线逻辑。
3. **版本控制**：全程接入 Git 并实时 Commit。

## 阶段零：Mock 数据与数据格式确立 (Phase 0: Data Schema & Mock) - [优先级：最高]
**目标**：确定与 AI 最终生成物兼容且能兼顾 Moonbit/Raylib 渲染性能的静态 3D 数据格式，并制造测试用的假数据。
- **任务 0.1**：确定 3D 渲染数据格式（点云/网格），兼顾 Raylib 渲染性能和 FFI 数据传输效率。
- **任务 0.2**：编写测试用的 Mock 场景数据文件（假坐标、假体素/点云数据、测试用 Shader 等）。

## 阶段一：核心引擎与基础渲染 (The Moonbit Client Core) - [优先级：最高]
**目标**：跑通 Moonbit 与 Raylib 的最简绑定，实现 3D 渲染环境。
- **任务 1.1**：搭建 Moonbit 项目结构，配置构建工具。
- **任务 1.2**：完成与 Raylib 的极简 C-FFI 绑定（窗口初始化、主循环、基础摄像机）。
- **任务 1.3**：支持加载并渲染静态 Mock 3D 数据（点云或基础网格）。
- **单元测试**：Moonbit 内存安全的 FFI 窗口创建与销毁测试。

## 阶段二：物理碰撞与游戏状态机 (Physics & State Machine) - [优先级：最高]
**目标**：在 Mock 数据下跑通 3D 漫游、实体碰撞与状态流转。
- **任务 2.1**：定义并实现健壮的 JSON 解析器（Moonbit 原生），解析 `world_data.json`。
- **任务 2.2**：实现底层碰撞系统，将离散的体素网格作为 Collider，处理实体碰撞（AABB 等）。
- **任务 2.3**：构建 Moonbit 核心游戏状态机（如 `Init` -> `Loading` -> `Roam` -> `Dialog`）。
- **单元测试**：状态机非法流转拦截测试；边界碰撞检测算法测试。

## 阶段三：游玩交互与体验打磨 (UI & Polish) - [优先级：高]
**目标**：实现完整的玩家交互反馈与 UI 层。
- **任务 3.1**：封装 Raylib 2D UI 接口，处理玩家触碰剧情节点后的对话框弹出。
- **任务 3.2**：实现跨语言日志系统（Log System）及可视化的 Debug 面板。
- **任务 3.3**：引入异步加载机制与场景过渡动画（防卡顿/防掉帧）。

## 阶段四：数据处理层重构 (Data Processing in Moonbit) - [优先级：中]
**目标**：将原本计划在 Python 端的数据降维、体素化等处理逻辑，尝试用 Moonbit 重写或封装。
- **任务 4.1**：用 Moonbit 实现从连续 3DGS 点云（或中间格式）到离散体素网格的“强制离散化”采样算法。
- **任务 4.2**：实现数据契约的严格校验与转换管道。

## 阶段五：AI 编译管线对接 (The AI Compiler) - [优先级：低（迭代更新）]
**目标**：预留接口，对接逐步迭代的 AI 资产生成管线。
- **任务 5.1**：对接 LLM 输出的场景大纲与结构化 JSON。
- **任务 5.2**：对接 SDXL 2D 出图与 DiffusionGS 3D 点云生成。

## 阶段六：系统健壮性与文档发布 (Docs & Release) - [优先级：中]
**目标**：交付最终文档与部署方案。
- **任务 6.1**：编写系统部署手册与构建指南。
- **任务 6.2**：完善 API 文档与 JSON 数据结构规范说明。
