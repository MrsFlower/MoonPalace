# Aetheria3D 引擎架构参考文档

本文档旨在说明 Aetheria3D 游戏引擎的核心架构设计、物理碰撞机制以及双渲染管线的工作原理。

## 1. 整体架构与语言边界

Aetheria3D 采用了 **MoonBit + C (Raylib)** 的跨语言融合架构：

- **C 层 (`src/raylib/raylib_wrapper.c`)**
  - **角色**：底层图形驱动封装与重活承担者。
  - **职责**：直接调用 Raylib 图形 API，管理 GPU 显存（模型、纹理贴图），编译并挂载 GLSL 着色器（如距离雾效果）。它还通过 C 函数导出了高效的体素数组访问接口，供 MoonBit 使用。
- **MoonBit 层 (`src/core_engine.mbt`, `src/voxel_world.mbt`)**
  - **角色**：高层业务逻辑与物理系统控制器。
  - **职责**：实现主游戏循环（Game Loop）、状态机切换（Loading/Roam/Dialog）、重力物理系统与包围盒碰撞检测。借助 MoonBit 的强类型系统和极速编译特性，大幅提升了游戏核心逻辑的迭代效率。

通过 Wasm-GC 相关的编译链和 FFI（Foreign Function Interface），两者实现了无缝的高性能通信。

## 2. 物理与碰撞系统设计

为了让玩家在这个体素世界中能够拥有稳定、无卡顿的物理体验（如跳跃、下落、沿墙滑行），我们对物理检测机制进行了特殊的解耦设计：

### 2.1 O(1) 的体素检测
我们摒弃了传统 3D 游戏常用的基于网格多边形的相交测试（Mesh Intersection），而是利用体素世界的离散特性。在 C 层提供了一个 `get_voxel_at(x, y, z)` 接口。
当 MoonBit 层传入连续的世界坐标时，C 层会通过反向偏移计算，将其直接映射到 3D 数组的整型索引上 `[grid_x, grid_y, grid_z]`，从而实现了 **O(1) 复杂度**的极速碰撞查询。

### 2.2 防弹跳 (Jitter) 与防卡死机制
早期的实现中，我们将玩家抽象为一个单一的包围盒（AABB），并同时用这个包围盒来处理所有方向的碰撞。这导致了一个致命缺陷：**脚底与地面的微小浮点误差，会被引擎误判为“卡入墙壁”，从而试图排斥玩家，引发无限上下弹跳（Jitter）**。

最终的设计方案：**严格分离水平与垂直检测**
- **垂直检测（重力与落地）**：引擎专门用玩家脚底的一个微小探测点（向下偏移 `0.499`，保留 Epsilon）来试探地面。一旦碰到底部方块，立即通过 `@math.floor` 反算出方块的绝对顶面高度，将玩家 Y 坐标“焊死”在这个高度上，并清零 Y 轴速度。
- **水平检测（墙壁滑行）**：检测身体撞墙时，**完全排除脚底点**，仅检测“膝盖”和“胸部”位置。如果 X 或 Z 方向发生碰撞，阻止该方向的位移，但允许另一方向继续运动（实现 Wall Sliding）。

## 3. 资产预处理与双渲染管线

由于生成式 AI 模型导出的 GLB 资产通常包含数十万级别的顶点，直接渲染或处理会有极大开销，因此项目设计了两套平行的渲染管线和资产处理工具链：

### 3.1 资产预处理 (`aetheria_foundry`)
- **体素化 (Voxelizer)**：在 `aetheria_voxelizer` 中，我们将原始 GLB 转化为粗颗粒度的体素 `.chunk` 文件，并在过程中使用**重心坐标钳制 (Barycentric Clamping)** 算法采样原贴图，消除了越界采样导致的“迷彩噪点”。
- **16-bit 索引溢出优化**：Raylib 默认仅支持 16-bit 的顶点索引（最大顶点数 65535）。为了在纯渲染管线中直接欣赏几十万顶点的高清模型，我们使用 Python 脚本 `optimize_glb_for_raylib.py` 提前将庞大的模型切分为多个安全的 Sub-mesh。如果不做此处理，模型将渲染为一团乱麻（破面）。

### 3.2 双编译入口 (Dual Entry Points)
得益于 MoonBit 轻量的包管理机制，我们通过两个独立的 `main.mbt` 入口，实现了两种运行时状态：

1. **体素物理管线 (`cmd/main.mbt`)**
   - 核心主线。加载体素 `.chunk` 构建隐形的物理世界，加载玩家控制器，同时在渲染时也可将原模型重叠渲染（通过按 `M` 键热切换 `render_mode`）。
   - 编译：`moon build --target native`，运行：`Aetheria3D.exe`

2. **GLB 纯渲染管线 (`cmd_glb/main.mbt`)**
   - 轻量级的展示路线。完全摒弃了体素加载和重力物理系统，只调用底层的 Raylib 接口加载优化后的 GLB 模型，并提供了一个自由视角的漫游摄像机（Free Camera）。适合在美术资产导入早期快速检查渲染质量和着色器效果。
   - 编译：`moon build --target native --package cmd_glb`，运行：`Aetheria3D_GLB.exe`
